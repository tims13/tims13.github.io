C++11:智能指针总结

发表于 2020-07-29 分类于 C++ 阅读次数：

ABOUT

参考：

Boost智能指针——weak_ptr

C++11中智能指针的原理、使用、实现

参考以上资料，总结一下C++11中的智能指针，加上自己的一些理解，方便查阅。

C++11引入智能指针的概念，是为了更好管理堆内存，减少内存泄漏的发生。一些需要注意的地方：

  1. 智能指针核心是运用了RAII（资源获取即初始化），对一个普通指针进行封装。
  2. 智能指针是一个类的对象，但是因为重构了操作符”->”与”*”，所以能够像一个指针那样用。

shared_ptr

shared_ptr使用了引用计数，多个指针指向一个对象的时候，shared_ptr通过管理引用计数，来达到管理内存的目的。

  1. 内部是线程安全的，但是对象的读取要加锁。
  2. 初始化的方式：传入指针到构造函数：shared_ptr<Node> sp(new Node());或者通过make_shared：shared_ptr<Node> sp = make_shared<Node> (node);。
  3. 利用原指针初始化最多一个shared_ptr，如果有多个，可能会导致多次释放同一段内存。
  4. 拷贝使被拷贝的对象的引用计数增加；赋值使原对象的引用计数减少（原对象被覆盖了）。

unique_ptr

顾名思义，unique_ptr唯一地指向其对象，同一时刻只能有一个unique_ptr指向一个对象。

  1. 作用域内，禁止赋值、禁止拷贝，只能转移。
  2. 利用RAII，异常的时候，动态资源能够利用析构函数进行释放。
  3. 如果离开作用域，unique_ptr将delete掉其指向的对象。

weak_ptr

weak_ptr为弱指针，因为其主要是为了配合shared_ptr来使用。

  1. 其没有重载”->”和”*”，所以其不具有普通指针的行为。
  2. 其可以从一个weak_ptr和shared_ptr来构造。
  3. 其没有共享资源的概念，作用是观测资源的使用情况。

循环引用

如果一个资源被循环引用，则会因为计数无法减少而导致资源无法释放（类似死锁），最终导致内存泄漏。

利用weak_ptr，可以很好地解决循环引用的问题，因为weak_ptr是不管理其指向内存的。
